## 0. 공통 규칙 (전 화면 적용)

- **하루 메인 기록 1개**: 하루에 메인 기록(사진+글)은 1개만 존재. 누구나 작성 가능하나 이미 있으면 댓글·반응만 가능. 작성 본인만 삭제 가능, 삭제 시 그날이 비워져 다른 사람이 새로 작성 가능.
- **본인 것만 수정·삭제**: 메인 기록·케어 기록·댓글 모두 작성한 본인만 수정·삭제. (Supabase RLS로 강제)
- **케어 순차 누적**: 밥·간식·산책은 시간대 고정 없이 누를 때마다 한 줄씩 누적. 각 줄에 닉네임 + 시간 표시.
- **케어 일일 리셋**: 케어 기록은 0시 기준. 홈에는 오늘 분만, 과거는 다이어리 상세에서 날짜별로.
- **반려동물 1마리 기준 UI(MVP)**: 화면은 가족의 대표 Pet 1마리 기준으로 동작. 단 모든 기록은 내부적으로 `pet_id`를 보유. 여러 마리는 2차에 "반려동물 전환" UI만 추가하면 확장됨(한 기록 = 한 마리).
- **이미지**: 업로드 시 클라이언트 압축(WebP), 변환 후 원본 삭제, 압축본+일러스트만 보관 (tech-stack.md 3.3).

## 1. 네비게이션 구조

```
App
├── (인증 전)
│   └── 로그인 (카카오 / 구글)
│
├── (온보딩 / 그룹 진입)  ← 탭 바깥, 풀스크린
│   ├── [최초 사용자] 반려동물 등록 → 가족 그룹 생성 → 초대 코드 발급
│   └── [초대받은 사용자] 초대 코드 입력 → 닉네임 선택/입력
│
└── (메인 / 하단 4탭)
    ├── 홈
    ├── 다이어리  ── (날짜 상세)  ← 푸시 화면
    ├── 일러스트 앨범  ── (월 폴더 → 그리드 → 일러스트 상세)
    └── 설정
    │
    └── (탭 위에 뜨는 화면)
        ├── 메인 기록 작성 / 수정
        ├── 케어 기록 추가 (간식·산책 메모 입력)
        └── (댓글·반응은 날짜 상세 내에서)
```

## 2. 진입 · 온보딩

### 2.1 로그인
- **목적**: 카카오/구글 간편로그인.
- **요소·동작**:
  - 앱 로고 + 한 줄 소개.
  - [카카오로 시작하기] → Supabase Auth 카카오 OAuth.
  - [구글로 시작하기] → Supabase Auth 구글 OAuth.
- **이동**:
  - 로그인 후 가족 그룹 소속이 **없으면** → 온보딩 분기 화면(그룹 생성 or 코드 참여 선택).
  - 이미 그룹이 **있으면** → 홈.

### 2.2 온보딩 분기
- **목적**: 새로 시작할지, 코드로 참여할지 선택.
- **요소·동작**:
  - [새로 시작하기 (반려동물 등록)] → 2.3
  - [초대 코드로 참여하기] → 2.4

### 2.3 최초 사용자 — 반려동물 등록 → 그룹 생성
- **목적**: 반려동물 프로필 만들고 가족 그룹 생성.
- **요소·동작**:
  - 프로필 사진(선택), 이름(필수), 종(선택), 생일·입양일(선택) 입력.
  - [완료] → 가족 그룹 자동 생성(작성자가 방장), Pet 레코드 생성, 초대 코드 발급.
- **이동**: 초대 코드 안내 화면 → [시작하기] → 홈.
- **메모**: 작성자는 자동으로 멤버 등록. 닉네임은 이후 설정에서 변경 가능(또는 이 단계에서 함께 받기).

### 2.4 초대받은 사용자 — 코드 입력 → 닉네임
- **목적**: 코드로 기존 그룹 참여 + 닉네임 지정.
- **요소·동작**:
  - 초대 코드 입력 필드 → [확인]으로 그룹 검증.
  - 닉네임 지정: 목록 선택(엄마/아빠/누나/형/동생/할머니/할아버지 등) + 직접 입력.
  - [참여하기] → 멤버로 등록.
- **이동**: 홈.

## 3. 홈 탭

### 3.1 목적
"오늘 하루"를 한 화면에. 반려동물 정보 + 오늘 케어(보기·입력) + 오늘 다이어리 미리보기.

### 3.2 와이어프레임 (요소별 동작 포함)

```
┌─────────────────────────────────────┐
│  [프로필사진]  콩이                   │  ← ① 반려동물 정보
│               2026년 6월 15일 (일)     │     프로필사진+이름+오늘날짜
├─────────────────────────────────────┤
│  오늘의 케어                          │  ← ② 케어 영역 (오늘 분만)
│                                       │
│  🍚 밥        [ + 추가 ]              │  · [+추가] 탭 → 즉시 한 줄 기록
│    · 엄마 · 08:10                     │     (밥은 메모 없이 바로)
│    · 아빠 · 13:20                     │  · 각 줄: 닉네임 · 시간
│                                       │  · 본인 줄만 길게눌러 수정/삭제
│  🦴 간식      [ + 추가 ]              │  · 간식/산책 [+추가] → 메모
│    · 엄마 · 15:00  "닭가슴살"          │     입력 시트(메모는 선택)
│                                       │
│  🐾 산책      [ + 추가 ]              │
│    (아직 없음)                        │  · 기록 없으면 빈 상태 문구
├─────────────────────────────────────┤
│  오늘의 일기                          │  ← ③ 오늘 다이어리 미리보기
│  ┌─────────────────────────────┐     │
│  │ [일러스트/사진 썸네일]        │     │  · 기록 있을 때:
│  │ "오늘 산책하다 만난 친구"      │     │     썸네일+글 일부 표시
│  │  - 엄마                       │     │     탭 → 다이어리 상세(3.x)
│  └─────────────────────────────┘     │
│   또는                                │  · 기록 없을 때:
│  ┌─────────────────────────────┐     │     "오늘 일기를 남겨주세요"
│  │  오늘 일기를 남겨주세요         │     │     탭 → 메인 기록 작성(5.1)
│  └─────────────────────────────┘     │
└─────────────────────────────────────┘
   [홈] [다이어리] [앨범] [설정]         ← 하단 탭
```

### 3.3 동작 정리
- **① 반려동물 정보**: 대표 Pet 1마리 표시(MVP). 탭 동작 없음(또는 프로필로 이동은 설정에서).
- **② 케어 영역**:
  - 밥 [+추가]: 즉시 기록(닉네임·현재시각). 메모 없음.
  - 간식/산책 [+추가]: 메모 입력 시트 → 저장. 메모는 선택(비워도 됨).
  - 각 줄은 본인 작성분만 수정·삭제.
  - 표시는 오늘 분만(0시 리셋). 과거는 다이어리 상세에서.
- **③ 오늘 다이어리 미리보기 (분기)**:
  - 오늘 메인 기록 **있음** → 썸네일+글 미리보기, 탭 시 **다이어리 상세**로.
  - 오늘 메인 기록 **없음** → "오늘 일기를 남겨주세요", 탭 시 **메인 기록 작성**으로.

## 4. 다이어리 탭

### 4.1 목적
과거 기록 탐색. 캘린더/리스트 두 보기를 토글 전환, 상세 화면은 공유.

### 4.2 보기 토글
- 상단에 [캘린더 | 리스트] 토글.
- **마지막 선택 보기는 사용자별로 저장** → 다음 진입 시 유지. (사용자 설정값으로 보관)

### 4.3 캘린더 보기
- 월 단위 캘린더.
- 각 날짜에 **기록·케어 유무를 점(dot)으로 표시**.
  - 메인 기록 있는 날 / 케어만 있는 날을 구분(예: 다른 색 점 또는 아이콘)하면 더 좋음.
- 날짜 탭 → **날짜 상세(4.5)**.

### 4.4 리스트 보기 (인스타 피드형)
- 최신순으로 카드 나열. 각 카드: **날짜 + 사진(일러스트/썸네일) + 글 일부**만 간단히.
- 카드 탭 → **날짜 상세(4.5)** (캘린더와 동일 화면).
- 메인 기록 없이 케어만 있는 날의 표시 방식은 캘린더와 일관되게 처리(피드에는 사진 있는 날 위주, 케어만 있는 날은 간소 카드 또는 생략 — 구현 시 택1, 기본은 간소 카드).

### 4.5 날짜 상세 (캘린더·리스트 공유)
- **목적**: 그날 하루 전체를 한 화면에.
- **구성(위→아래)**:
  1. 날짜 헤더.
  2. **메인 기록**: 사진(일러스트/압축본 전환 보기 가능하면 좋음) + 글 + 작성자.
     - 본인 작성이면 수정/삭제 진입.
  3. **케어 내용**: 그날의 밥·간식·산책 기록 전체(닉네임·시간·메모).
  4. **댓글·반응**: 댓글 목록 + 입력창, 반응(좋아요/이모지). 본인 댓글만 수정·삭제.
- **빈 날**: 메인 기록 없으면 "이 날의 일기가 없어요" + (오늘이면 작성 버튼). 케어만 있으면 케어 섹션만 표시.

### 4.6 와이어프레임 — 날짜 상세 (요소별 동작 포함)

```
┌─────────────────────────────────────┐
│ ←  2026년 6월 15일 (일)               │  ← 뒤로가기 + 날짜 헤더
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐     │  ← ① 메인 기록
│  │   [일러스트 / 사진]           │     │  · 사진 영역(탭: 크게보기)
│  │                             │     │  · (가능 시) 일러스트⇄원본 토글
│  └─────────────────────────────┘     │
│  "오늘 산책하다 친구를 만났다"          │  · 본문 글
│   - 엄마 · 오후 7:24                   │  · 작성자 닉네임·시각
│                          [⋯]          │  · 본인 작성분이면 ⋯ → 수정/삭제
├─────────────────────────────────────┤
│  케어 기록                            │  ← ② 케어 내용(그날 전체)
│  🍚 밥   · 엄마 08:10  · 아빠 13:20   │  · 종류별 줄 나열
│  🦴 간식 · 엄마 15:00 "닭가슴살"        │  · 닉네임·시간·메모
│  🐾 산책 · 아빠 19:00 "공원 30분"       │  · 본인 줄만 수정/삭제
├─────────────────────────────────────┤
│  반응  ♥ 3                            │  ← ③ 반응
│  [♥ 좋아요]                           │  · 탭 토글
│                                       │
│  댓글                                 │  ← ④ 댓글
│  · 아빠: 너무 귀엽다 ㅎㅎ              │  · 목록(닉네임+내용)
│  · 누나: 산책 같이 갈걸               │  · 본인 댓글만 수정/삭제
│  ┌─────────────────────────┐ [전송]  │  · 입력창 + 전송
│  │ 댓글 입력…               │         │
│  └─────────────────────────┘         │
└─────────────────────────────────────┘
```

## 5. 기록 작성 화면

### 5.1 메인 기록 작성
- **진입**: 홈의 "오늘 일기를 남겨주세요", 또는 다이어리 빈 날(오늘)의 작성 버튼.
- **요소·동작**:
  - 사진 선택(갤러리/카메라) → 클라이언트 압축.
  - 글 입력.
  - [저장] → 압축본 업로드 + 일러스트 변환 트리거(Edge Function) → 저장.
  - 변환 진행 중 로딩/플레이스홀더, 실패 시 재시도.
- **가드**: 저장 시점에 이미 오늘 메인 기록이 존재하면(동시성) 안내하고 댓글·반응으로 유도.
- **이동**: 저장 후 다이어리 상세 또는 홈.

### 5.2 메인 기록 수정
- 본인 작성분만. 글 수정 중심. (사진 교체 허용 여부는 구현 시 결정 — 교체 시 재변환)

### 5.3 케어 기록 추가
- **밥**: [+추가] 즉시 기록(메모 없음).
- **간식·산책**: [+추가] → 메모 입력 시트(선택) → 저장.
- 모든 케어 줄은 본인 작성분만 수정·삭제.

## 6. 일러스트 앨범 탭

### 6.1 목적
변환된 일러스트를 **연·월 폴더**로 모아보기.

### 6.2 폴더 목록 (1단계)
- 연·월 폴더를 **최신순**으로 나열.
- 각 폴더 표지: **대표 썸네일(그 달 첫 장) + "2026.06" + 총 장수**.
- **기록 있는 달만** 폴더 생성(빈 달은 표시 안 함).
- 폴더 탭 → 그리드(6.3).

### 6.3 월 그리드 (2단계)
- 아이폰 앨범처럼 **가로 3열 그리드**.
- 정렬: 최신이 위로.
- 항목 탭 → 일러스트 상세(6.4).

### 6.4 일러스트 상세 (3단계)
- 일러스트 크게 보기.
- 동작: 저장(다운로드)·공유, "이 일러스트의 다이어리로 이동"(해당 날짜 상세로).

### 6.5 와이어프레임 (요소별 동작 포함)

```
[1단계: 폴더 목록]            [2단계: 월 그리드]         [3단계: 상세]
┌───────────────────┐        ┌───────────────────┐     ┌───────────────┐
│ 일러스트 앨범       │        │ ← 2026.06          │     │ ←     [저장][공유]│
│ ┌────┐ 2026.06    │        │ ┌──┐┌──┐┌──┐       │     │  ┌─────────┐  │
│ │썸네일│ 12장  >   │ ─탭→   │ └──┘└──┘└──┘       │ ─탭→│  │ 일러스트  │  │
│ └────┘            │        │ ┌──┐┌──┐┌──┐       │     │  └─────────┘  │
│ ┌────┐ 2026.05    │        │ └──┘└──┘└──┘       │     │  [다이어리로 →]│
│ │썸네일│ 30장  >   │        │  (3열, 최신순)      │     │               │
│ └────┘            │        │                    │     │               │
└───────────────────┘        └───────────────────┘     └───────────────┘
 · 최신순                      · 가로 3개씩            · 저장/공유
 · 표지=첫장+연월+장수          · 탭→상세              · 해당 날짜 상세로 이동
```

## 7. 설정 탭

### 7.1 MVP 포함
- **가족 그룹**
  - 초대 코드 보기 · 공유(카톡 등으로 전달).
  - 가족 멤버 목록(닉네임).
  - 내 닉네임 변경.
  - 그룹 나가기.
- **반려동물 프로필**

## 8. 데이터 ↔ 화면 매핑 메모

- **Pet**: 자기 PK(id) 보유, `family_id`로 가족 소속. 화면은 대표 Pet 1마리 사용(MVP).
- **DiaryEntry**: 자기 PK(id), `pet_id`(FK)로 반려동물 참조, 날짜별 1개. → 홈 미리보기 / 다이어리 상세.
- **CareLog**: 자기 PK(id), `pet_id`(FK), 종류(밥/간식/산책)·작성자·시간·메모. → 홈 케어 / 상세 케어.
