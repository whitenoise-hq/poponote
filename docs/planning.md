# 포포노트 (PopoNote) — 기획서 (planning.md)

> 가족이 함께 쓰는 반려동물 다이어리
> v1.1 · 2026-06-15

---

## 1. 개요

### 1.1 앱 이름
- 국문: **포포노트**
- 영문: **PopoNote**
- 발바닥 '포'에서 착안한 귀여운 어감 + '노트'로 기록·다이어리 성격을 직관적으로 전달

### 1.2 한 줄 정의
가족이 초대 코드 하나로 함께 들어와, 하루 한 장씩 반려동물의 사진·일기를 쌓고 밥·간식·산책을 같이 챙기는 다이어리 앱.

### 1.3 핵심 가치
- **함께 본다**: 가족이 같은 기록을 보고 댓글·반응으로 참여
- **부담 없다**: 하루 한 장이면 충분, 일기를 못 쓴 날도 케어 기록으로 채움
- **남는다**: 사진이 일러스트로 변환되어 예쁜 앨범으로 쌓임
- **헷갈리지 않는다**: 밥·간식·산책을 누가·언제 했는지 보여 중복 급여를 방지

### 1.4 타깃
- 가족 단위(2인 이상)로 반려동물을 키우는 집
- "누가 밥 줬어?"를 매일 묻는 가정, 떨어져 사는 가족과 소식을 나누고 싶은 사람

## 2. 핵심 규칙

### 2.1 하루 기록 단위
- 하루에 **메인 기록(사진 + 글)은 1개**만 존재한다.
- 메인 기록은 **가족 누구나** 작성할 수 있으나, 하루에 한 번만 작성된다.
- 이미 메인 기록이 있으면 나머지 가족은 **댓글·반응만** 가능하다.
- 메인 기록은 **작성한 본인만** 삭제할 수 있다.
- 삭제하면 그날 기록이 다시 비워지고, **다른 가족이 새로 작성**할 수 있다.

### 2.2 케어 기록 (밥·간식·산책)
- 끼니를 시간대로 고정하지 않는다. **누를 때마다 순차적으로 쌓이는** 방식.
- 밥: 탭하면 바로 한 줄 추가.
- 간식·산책: 탭하면 메모(무엇을/어디를) 입력 가능, 메모는 선택.
- 각 기록 줄에는 **닉네임 + 입력 시간**이 표시된다.
- 케어 기록은 **하루(0시) 기준으로 리셋** — 홈에는 오늘 분만, 과거는 다이어리에서 날짜별로 본다.

### 2.3 수정·삭제 권한
- **본인이 작성한 것만** 수정·삭제 가능. (메인 기록, 케어 기록, 댓글 모두 동일)

### 2.4 닉네임
- 코드로 참여할 때 닉네임 지정.
- **목록 선택**(엄마, 아빠, 누나, 형, 동생, 할머니, 할아버지 등) **+ 직접 입력** 둘 다 지원.

## 3. 사용자 흐름

### 3.1 최초 사용자(방장)
1. 앱 실행 → 반려동물 프로필 등록(이름, 종, 생일 등)
2. 가족 그룹 자동 생성 → 초대 코드 발급
3. 코드를 가족에게 공유
4. 첫 기록 작성

### 3.2 초대받은 가족
1. 앱 실행 → "코드로 참여"
2. 초대 코드 입력
3. 닉네임 선택/입력
4. 바로 그룹의 기록 열람·참여

### 3.3 일상 사용(매일)
1. 홈에서 오늘 상태 확인(케어 현황, 오늘 일기 유무)
2. 밥·간식·산책 케어 기록 추가
3. 메인 기록 작성(없을 때) 또는 댓글·반응(있을 때)
4. 사진은 자동으로 일러스트 변환되어 앨범에 저장

## 4. 기능 명세

### 4.1 메인 기록 (사진 + 글)
- 사진 1장 + 텍스트
- 작성 시각·작성자(닉네임) 기록
- 본인만 수정·삭제

### 4.2 일러스트 변환
- 업로드한 사진을 일러스트 스타일로 자동 변환
- **변환 후 원본은 삭제**, 화면 표시용 **압축본 + 일러스트**를 보관 (상세는 tech-stack.md 3.3)
- 앨범에는 일러스트 노출
- 변환 실패 시 재시도 가능
- MVP는 변환 스타일 **1종 고정** (선택형은 추후)

### 4.3 케어 기록 (밥·간식·산책)
- 항목별 "추가" 동작 → 그 항목 아래에 한 줄씩 누적
- 줄마다 닉네임 + 시간 표시
- 간식·산책은 메모 선택 입력
- 본인 기록만 수정·삭제
- 확장 여지: 물·약 등 항목 추가, 산책 시간·거리 (추후)

### 4.4 댓글 / 반응
- 메인 기록에 댓글(텍스트) + 간단 반응(좋아요/이모지)
- 본인 댓글만 수정·삭제

### 4.5 가족 그룹 / 초대 코드
- 그룹당 초대 코드 1개
- 코드로 참여 → 닉네임 지정
- 모든 멤버가 기록·케어·댓글 참여 가능 (권한은 "본인 것만 수정/삭제"로 단순화)

## 5. MVP 범위

### 5.1 1차 출시 포함
- 반려동물 프로필 등록
- 가족 초대 코드 / 코드로 참여 / 닉네임
- 메인 기록(사진+글) 작성·열람, 본인 수정·삭제
- 사진 → 일러스트 변환 + 앨범
- 케어 기록(밥·간식·산책, 누가/언제, 순차 누적)
- 댓글·반응
- 4탭 네비게이션(홈 / 다이어리 / 일러스트 앨범 / 설정)

### 5.2 2차 이후
- 푸시 알림(일기 없을 때, 새 댓글)
- 기념일(생일·입양일) 등록·알림
- 월별 일러스트 콜라주/회고
- 반려동물 여러 마리 지원 (데이터 구조는 미리 대비)
- 통계(케어 패턴, 활동 요약)
- 케어 항목 확장(물·약), 산책 시간·거리

## 7. 데이터 구조 초안

- **Family(그룹)**: id, 초대코드, 생성일, 방장
- **Member**: id, family_id, 닉네임, 참여일
- **Pet**: id, family_id, 이름, 종, 생일, 입양일, 프로필사진
- **DiaryEntry(메인 기록)**: id, family_id, pet_id, 날짜, 작성자, 글, 압축사진(URL), 일러스트(URL), 생성/수정 시각
- **CareLog(케어)**: id, family_id, pet_id, 날짜, 종류(밥/간식/산책), 작성자, 시간, 메모
- **Comment**: id, entry_id, 작성자, 내용, 시각
- **Reaction**: id, entry_id, 작성자, 종류
